# 转盘抽事件改造方案

> 状态：方案已确认，待开发  
> 记录日期：2026-07-16  
> 对应当前版本：`0.3.2-internal`

## 一、改造结论

下一轮玩法改造采用“转盘抽事件 + 月份时间线”方案。

- 转盘取代游戏页中的骰子、路径和棋子，负责决定本月抽到哪一类事件。
- 月份时间线负责表达时间推进，每次转盘固定推进 1 个月。
- 首页海报和最终结果页仍可保留人生路径，作为品牌视觉和旅程回顾元素。
- 现有事件效果、现金流结算、卡牌冷却、最大出现次数、长期计划和结果页计算不在本轮重做。

当前版本仍使用骰子与曲线路径。本文档只沉淀下一轮方案，尚未接入正式游戏。

## 二、为什么改成转盘

当前骰子点数只改变棋子落到哪个节点，但玩家无法选择路线，也不能调整步数。玩家实际体验更接近“点击后随机抽一张事件”，骰子的 1-3 点没有形成策略价值。

转盘的优势：

- 随机过程更直观，指针停靠和翻牌能形成短暂期待。
- 不再暗示玩家可以控制路线或步数。
- 能直接复用现有六类事件卡池，不需要重做财务规则。
- 主游戏页可以移除较大的地图背景和棋子资源，进一步改善手机端加载速度。
- 每次操作固定推进 1 个月，时间规则更容易解释。

不采用“几张可选卡背”作为主玩法，原因是：

- 如果显示卡牌类型，玩家会倾向避开风险，改变当前以随机为主的规则。
- 如果不显示类型，选择哪张卡没有差别，只是把点击骰子换成点击卡背。

卡背可以保留在转盘中央，作为翻牌动效的视觉元素。

## 三、页面结构

游戏页从上到下调整为：

```text
当前进度、更多选项
现金储备、安全垫
月收入、月支出

本月事件转盘
六种事件区域 + 中央指针/卡背

月份时间线
1月 ●—●—●—○—○ 12月

转动转盘
人生日志
```

原地图页中的以下内容从主游戏区移除：

- 曲线路径；
- 地图节点；
- 玩家棋子；
- 骰子点数；
- “掷骰前进”按钮。

地图下方正在生效的长期计划、临时支出和保障状态继续保留，放在转盘卡片下方。

## 四、转盘事件类型

转盘直接对应现有六类卡牌，避免给所有事件新增字段。

| 转盘区域 | 建议颜色 | 现有事件分类 | 初始占比 |
| --- | --- | --- | ---: |
| 生活支出 | 橙色 | `expense_up` | 22% |
| 突发支出 | 红色 | `one_time_cost` | 13% |
| 收入波动 | 暖灰色 | `income_down` | 17% |
| 收入与好运 | 绿色 | `positive` | 22% |
| 健康与保障 | 蓝色 | `health_risk` | 13% |
| 人生选择 | 紫色 | `choice` | 13% |

这些占比来自当前 16 个地图节点中各分类的大致出现比例，用于迁移现有随机结构，不是新增的保赢或保输机制。

“低估区域的定投机会”仍属于 `choice`，不单独制作投资区域。当前只有少量投资卡，如果给它一个固定扇区，会不合理地放大投资卡出现频率。

## 五、随机抽取顺序

一次转盘应按照以下顺序处理：

1. 根据当前月份、身份状态、卡牌最大次数、冷却期、长期计划和后续状态，整理本回合可用卡牌。
2. 按六种事件类型分组，移除当前没有可用卡牌的类型。
3. 按转盘基础占比抽取一个事件类型。
4. 在该类型的可用卡牌中，继续使用现有权重随机抽取具体卡牌。
5. 根据最终卡牌类型计算指针停止角度。
6. 播放指针旋转和卡牌翻开动效。
7. 普通事件直接结算；选择事件显示居中的选项窗口。

必须先确定卡牌，再播放转盘动画。这样能保证指针停靠颜色和实际翻开的事件类型完全一致。

内部测试模式指定下一张卡牌时，转盘应根据指定卡牌类型停到对应区域，不重新随机卡牌。

如果某一分类暂时没有符合条件的卡牌，只在本次抽取中重新归一化其他分类的概率，不补发卡牌，也不绕过冷却规则。

## 六、转盘视觉与动效

视觉继续使用当前“轻桌游金融卡片风”：

- 奶油纸面背景；
- 深褐黑标题；
- 六种低饱和事件颜色；
- 柔和边框和阴影；
- 转盘放在独立奶油白卡片中；
- 中央放事件卡背或转轴；
- 不使用外部游戏引擎或 Canvas。

建议使用 HTML/CSS 实现六个扇区，或使用内联 SVG 绘制扇区；指针使用 CSS `transform` 旋转。

动效节奏：

- 点击后按钮立即锁定，避免重复触发；
- 指针旋转 3-5 圈；
- 总时长控制在 1-1.3 秒；
- 使用明显减速曲线停靠；
- 停靠区域短暂发光或轻微上浮；
- 中央卡牌用约 250ms 完成翻牌；
- 从点击到看到事件尽量控制在 2 秒以内。

开启系统“减少动态效果”时，不旋转多圈，只保留短暂转向、区域高亮和卡牌淡入。

## 七、月份时间线

转盘不再表达前进步数，因此时间感由月份时间线承担。

统一规则：

- 每次转动只推进 1 个月；
- 当前月份使用橙色高亮；
- 已完成月份使用实心节点；
- 未来月份使用浅色空心节点；
- 顶部继续显示“第 X / N 个月”。

不同挑战长度：

- 12个月：完整显示 12 个节点。
- 24个月：分为第1年和第2年，每年 12 个节点。
- 36个月：分为第1年、第2年和第3年，每年 12 个节点。

进入新年度时可以显示一次很短的阶段提示，但不增加确认按钮。

## 八、事件处理流程

普通事件：

```text
转动转盘
→ 指针停靠
→ 卡牌翻开
→ 系统直接结算
→ 居中结果卡
→ 点击卡片外关闭
→ 进入下一月
```

选择事件：

```text
转动转盘
→ 指针停靠
→ 卡牌翻开
→ 居中选择窗口
→ 玩家选择
→ 居中结果卡
→ 点击卡片外关闭
→ 进入下一月
```

不恢复普通事件的“确认影响”按钮，也不增加第二次继续按钮。

## 九、数据与旧存档兼容

为避免旧存档损坏：

- 继续兼容读取 `position` 和 `lastDice`，但新界面不再使用；
- 不删除旧地图字段，等版本稳定后再做数据迁移；
- `currentMonth`、`completedMonths` 和 `maxMonth` 继续作为时间进度唯一口径；
- 卡牌历史、月度快照和结果页读取方式不变；
- 调试模式的固定种子和指定事件继续可用。

建议新增状态：

```js
wheelState: {
  isSpinning: false,
  lastSector: null,
  rotation: 0
}
```

其中旋转角度只用于界面，不参与财务计算，也不需要长期写入存档。

## 十、埋点调整

新增：

- `cash_game_wheel_spun`：玩家点击转盘；
- `cash_game_wheel_landed`：记录停靠分类和月份；
- `cash_game_card_revealed`：卡牌完成翻开。

逐步停用：

- `cash_game_dice_rolled`。

需要继续保留：

- 卡牌处理；
- 选择结果；
- 月份结算；
- 定投、保障和卖出；
- 完成或提前结束。

## 十一、开发顺序

### 第一阶段：规则迁移

- 抽取独立的“获取本回合可用卡牌”函数。
- 建立六类转盘配置和概率。
- 将当前按地图位置选卡改为按转盘类型选卡。
- 保证冷却、最大次数、定投窗口和指定事件逻辑不变。

### 第二阶段：转盘组件

- 实现六色转盘和中央指针。
- 实现停止角度计算、旋转锁定和减少动态效果。
- 将“掷骰前进”替换为“转动转盘”。

### 第三阶段：时间线与页面整理

- 增加 12/24/36 个月时间线。
- 移除主游戏页地图背景、路径、棋子和骰子展示。
- 保留长期计划、临时支出、人生日志和菜单。
- 更新玩法说明中的骰子和地图文案。

### 第四阶段：测试与文档

- 补充转盘概率和分类一致性测试。
- 完整测试 12、24、36 个月。
- 更新 README、内部测试指南和埋点说明。
- 对比手机端首屏资源和交互耗时。

## 十二、验收标准

功能验收：

- 指针停靠颜色与抽到的卡牌类型 100% 一致。
- 每次转动最多处理一张卡牌、结算一次、推进一个月。
- 快速连点不会重复抽卡或重复扣款。
- 普通事件不需要确认影响。
- 选择事件、保障、定投和后续事件正常生效。
- 12、24、36 个月均可正常完成或提前结束。
- 旧存档可以继续进入游戏。

体验验收：

- 玩家不用解释就能理解“转一次，过一个月”。
- 点击后约 2 秒内看到事件。
- 转盘动画不会成为每回合的等待负担。
- 手机端没有横向滚动、文字重叠或难以点击的区域。
- 普通回合关闭结果后可以立即进行下一月。

数值验收：

- 大量固定种子模拟中，六类事件占比接近配置值。
- 卡牌冷却和最大出现次数继续有效。
- 不为了短局强制出现定投、估值修复、阶段高估或其他剧情卡。
- 现金储备、月度快照、人生日志和最终结果保持一致。

## 十三、本轮不做

- 不增加路线分支；
- 不增加重转、控制指针或修改概率的道具；
- 不加入精力值、行动点或排行榜；
- 不重做投资净值线；
- 不增加强制保底事件；
- 不为了转盘补充大量新卡牌；
- 不修改首页海报的路径视觉。
