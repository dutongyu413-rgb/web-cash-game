# 身份卡数据

身份卡用于决定玩家开局的现金流起点。当前版本将身份卡简化为“职业角色卡”，只保留会直接影响游戏结果的字段。

## 字段约定

| 字段 | 含义 | 是否展示 | 是否影响游戏 |
|---|---|---|---|
| `id` | 身份卡唯一标识 | 否 | 是，用于选择和开始游戏 |
| `name` | 职业角色名 | 是 | 是，作为玩家身份名称 |
| `income` | 常规月收入 | 是 | 是 |
| `expense` | 常规月支出 | 是 | 是 |
| `savings` | 初始现金储备 | 是 | 是 |

当前不使用这些字段：

| 字段 | 移除原因 |
|---|---|
| `age` | 暂不影响游戏机制 |
| `job` | 和 `name` 重复，统一用职业角色名表达 |
| `family` | 暂不影响游戏机制 |
| `description` | 暂不影响游戏机制，且容易变成说明文字堆叠 |

身份页不展示初始安全垫，避免玩家只为了抽到更高安全垫而反复重抽。安全垫仍会在正式游戏过程中展示和结算。

## 调整原则

- 职业名尽量清楚、生活化，避免同时出现“身份”和“职位”两套说法。
- 每张身份卡的差异应主要体现在收入、支出、现金储备，而不是描述文案。
- 月收入、月支出、现金储备需要贴近用户直觉，不要为了难度刻意极端化。
- 可以保留不同压力区间，便于测试不同局面：
  - 低收入低支出：如图书管理员、教师。
  - 中等收入中等支出：如企业白领、数据分析师、销售。
  - 高收入高支出：如医生、餐饮老板、运动员。
  - 高收入较稳储备：如高级工程师、程序员。
- 如果未来要加入年龄、家庭、行业风险等信息，建议先把它们转化成明确机制，例如固定支出、收入波动、事件权重或专属事件池，再放回身份卡。

## 数据维护位置

当前实际数据在 [script.js](./script.js) 的 `identityCards` 数组中。

如果修改本文件中的表格，也需要同步修改 `script.js`，否则游戏不会读取到新数据。
