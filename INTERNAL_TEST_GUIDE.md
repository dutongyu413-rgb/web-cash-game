# 内部测试指南

## 测试目标

本轮测试重点不是判断游戏是否“足够好玩”，而是先确认：新玩家能否独立完成一局、财务变化是否可信、选择是否真实生效、问题是否可以复现。

当前内部测试版本：`0.3.0-internal`。

## 测试入口

普通试玩：

```text
http://127.0.0.1:4173/
```

内部测试模式：

```text
http://127.0.0.1:4173/?debug=1&seed=your-name
```

进入游戏后，地图右上角会出现 `TEST`。测试面板可以：

- 设置随机种子。
- 修改当前月份和现金储备。
- 指定下一张事件卡。
- 重置新手引导。
- 复制本局测试诊断。

这些能力只在调试链接中出现，不影响普通玩家。

## 三类测试路线

### A. 第一次玩的同事

1. 不做任何说明，直接打开普通试玩链接。
2. 观察首页、身份选择和三步引导是否足以理解玩法。
3. 至少完成 6 个月。
4. 记录第一次不知道该点哪里、看不懂数字或不理解结算的时刻。

通过标准：不需要口头指导即可完成开局、掷骰、处理卡牌和进入下个月。

快速回合专项检查：

1. 新开局默认选择12个月，仍可改选24或36个月。
2. 普通事件不再要求点击“确认影响”，处理后显示居中的回合结果卡；结果卡不会自动消失，点击卡片外的遮罩后返回地图。
3. 快速反馈只展示发生变化的项目，关闭后可以立即继续掷骰。
4. 第4、8个月显示阶段完成提示，但不增加确认按钮。
5. 保障实际减损在结果卡中单独强调；现金流击穿仍显示完整结算。

### B. 完整游玩测试

1. 完整完成一局 12 个月。
2. 再选择 24 或 36 个月，至少玩到第 12 个月。
3. 检查一次性影响、阶段性影响和永久影响是否符合卡牌描述。
4. 检查地图、结算、人生日志和结果页的金额是否相互一致。

通过标准：没有阻塞流程的问题，玩家能够说清本局为什么赢或为什么提前结束。

### C. 规则专项测试

使用内部测试面板逐项指定卡牌：

- `运动受伤`：分别在有保障和无保障时测试。
- `低估区域的定投机会`：测试不参加、开始定投、卖出一半和全部卖出。
- 普通随机局不应在第 10 个月强制定投机会；市场节点按自然月份推进，12 个月局不保证出现估值修复，也不应为了结算强制出现阶段高估。
- `职业提升课程`：跳转后续月份，测试成功和未立即生效的回响。
- `是否换车`：检查现金储备、月供和隐藏后续事件。
- `公司业务调整`：检查实际生效月份和结果页累计冲击。
- `带家人短途旅行`：检查立即出行和推迟后的现金变化。

通过标准：卡牌、月度结算、地图状态、人生日志和结果页使用同一套实际数据。

## 发布前检查

- 首页图片在手机网络下可以正常出现，按钮可点击。
- 首次新手引导只自动展示一次，菜单可以重新打开。
- 12、24、36个月均可正常开局和结束。
- 现金储备低于0时进入结果页，并显示实际完成月份。
- 一次性事件不修改常规月收入或月支出。
- 临时效果按描述月份结束，永久效果不会意外恢复。
- 保障减免同时体现在事件卡、本月结算和最终结果。
- 卖出投资后现金储备增加，全部卖出后不再显示进行中的定投。
- 主动结束、重新开始和继续存档都不会卡住。
- 390px和430px宽度下没有文字重叠、按钮溢出或无法滚动的弹窗。
- 浏览器控制台没有未处理错误。
- `npm test` 全部通过。
- `npm run simulate` 已重新生成数值基线。

## 反馈分级

| 等级 | 定义 | 示例 |
| --- | --- | --- |
| P0 | 无法继续游戏或存档损坏 | 按钮无反应、结算无法关闭、刷新后状态错乱 |
| P1 | 金额或规则明显错误 | 一次性支出变成永久月支出、卖出后储备未增加 |
| P2 | 明显影响理解或操作 | 看不懂下一步、重要状态没有反馈、手机文字重叠 |
| P3 | 体验优化建议 | 动效节奏、颜色、措辞和细节质感 |

优先修复顺序为 P0、P1、P2。P3 先归档，避免测试期间不断改变视觉基线。
